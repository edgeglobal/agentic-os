# Team-Wissen — INDEX

> Operating know-how. Wie das Team arbeitet, nicht was es weiß über das Geschäft.

## Struktur

| Ordner | Inhalt |
|---|---|
| **SOPs** | Standardabläufe — atomar, ein Job pro Datei. Filename: `SOP-NNN-<titel>.md` |
| **Workflows** | Multi-Spezialist-Orchestrierungen. Filename: `WF-NNN-<titel>.md`. Referenzieren SOPs/Richtlinien, duplizieren sie nie. |
| **Richtlinien** | Statische Referenz-Info (Naming, Tonalität). Filename: `R-NNN-<titel>.md`. SOPs/Workflows verlinken via `[[wikilink]]` darauf. |
| **Session-Logs** | Append-only Sitzungs-Gedächtnis. `YYYY/MM/YYYY-MM-DD-<slug>.md` |

## Onboarding-Prozess für die Wissensbasis

Wenn ihr neue SOPs/Workflows/Richtlinien anlegt:

1. Datei nach Schema benennen (`SOP-004-...`, etc.)
2. In den passenden Ordner ablegen
3. **Hier in INDEX.md eintragen** (sonst fehlt die Auffindbarkeit)
4. In passenden anderen Dateien per `[[wikilink]]` verweisen

Larry erzwingt das am Session-Ende als Bibliothekar.
